# How to Add `dc.rights.holder` Field in DSpace JSPUI

This guide provides step-by-step instructions to configure and display the `dc.rights.holder` metadata field in **DSpace JSPUI**. It includes:

- Displaying the field in item view
- Customizing the field label
- Adding the field to the submission form

---

### 🛠 Step 1: Edit Configuration File

Open the following file:

```
[dspace]/config/dspace.cfg
```

Update the following property to include `dc.rights.holder`:

```properties
webui.itemdisplay.default = dc.title, dc.contributor.author, dc.date.issued, dc.rights.holder
```
---
### 🔄 Step 2: Restart Tomcat

```bash
sudo systemctl restart tomcat9
```

---


### 📝 Step 3: Modify Messages File

Edit the following file:

```
[dspace]/jspui/src/main/resources/messages.properties
```

Add or update this line:

```properties
metadata.dc.rights.holder = Rights
```

This label will be shown on the item view instead of the raw metadata key.
---

### 🧱 Step 4: Rebuild and Deploy JSPUI

```bash
cd [dspace]/jspui
mvn package

cp target/dspace-jspui.war [tomcat]/webapps/
sudo systemctl restart tomcat9
```

---

### ✍️ Step 5: Edit Submission Form Configuration

Open the file:

```
[dspace]/config/input-forms.xml
```

Find the `<form-definition name="traditional">` block, and within a `<page>` block, add the following:

```xml
<field>
    <dc-schema>dc</dc-schema>
    <dc-element>rights</dc-element>
    <dc-qualifier>holder</dc-qualifier>
    <label>Rights Holder</label>
    <input-type>onebox</input-type>
    <required>false</required>
    <repeatable>false</repeatable>
    <hint>Enter the name of the rights holder (e.g., author, institution)</hint>
</field>
```

### 📌 Step 6: Ensure Proper Form Mapping

At the bottom of `input-forms.xml`, confirm this mapping exists:

```xml
<form-map>
    <name-map collection-handle="default" form-name="traditional"/>
</form-map>
```

### 🔄 Step 7: Restart Tomcat Again

```bash
sudo systemctl restart tomcat9
```

---

## ✅ Result

- `dc.rights.holder` is visible in item view with a friendly label.
- It can be input during item submission.
- The field label is customizable through `messages.properties`.

---

## 📎 Notes

- Ensure the field is registered in the metadata registry:  
  ```
  [dspace]/config/registries/dublin-core-types.xml
  ```
- You can adjust `required` or `repeatable` settings in the form.
- For multilingual labels, update `messages_XX.properties`.
